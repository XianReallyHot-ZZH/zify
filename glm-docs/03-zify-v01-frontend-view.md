# Zify 一期前端信息架构

> 基于一期 7 个功能模块，设计前端页面结构和导航体系。

---

## 导航结构

采用左侧导航栏 + 内容区的经典布局。共 7 个一级导航项，按用户使用频率排序：

```
┌──────────┬─────────────────────────────────────────┐
│ 左侧导航  │              内容区                       │
│          │                                         │
│ 对话      │                                         │
│ Agents   │                                         │
│ 工作流    │                                         │
│ 知识库    │                                         │
│ 工具      │                                         │
│ 模型管理   │                                         │
│          │                                         │
└──────────┴─────────────────────────────────────────┘
```

## 页面清单

### 1. 对话

用户日常使用的核心页面，也是产品的默认落地页。

```
/ (默认页)
├── 左栏：会话列表
│   ├── 按 Agent 分组展示历史会话
│   ├── 新建对话按钮 → 弹出 Agent 选择器
│   └── 搜索历史会话
│
└── 右栏：对话区
    ├── Agent 名称 + 状态
    ├── 消息流（用户消息、AI 回复、工具调用过程）
    ├── 输入框（支持粘贴文本）
    └── 中断按钮（ReAct 循环中可中断）
```

**说明**：
- 进入页面时若没有活跃会话，显示引导用户选择 Agent 开始对话
- 选择 Agent 后自动创建新会话
- 点击左侧历史会话可继续对话
- 不需要独立的 Agent 选择页面，对话页内完成所有操作

---

### 2. Agents

Agent 的管理页面。

```
/agents
├── /agents                     # Agent 列表页
│   ├── 搜索框
│   ├── 新建 Agent 按钮
│   └── Agent 卡片列表（名称、描述、类型标签、最近对话时间）
│
└── /agents/create              # 创建 Agent（分步表单）
或 /agents/[id]/edit            # 编辑 Agent
    ├── Step 1: 基础信息
    │   ├── 名称
    │   ├── 描述
    │   └── 类型选择：ReAct Agent / Workflow Agent
    │
    ├── Step 2: 人设配置
    │   └── System Prompt 编辑器
    │
    ├── Step 3: 能力配置
    │   ├── 模型选择（下拉，来自模型管理）
    │   ├── 工具绑定（多选，来自工具管理）
    │   └── 知识库绑定（多选，来自知识库）
    │       （Workflow Agent 不显示此步，改为绑定工作流）
    │
    └── Step 4: 确认保存
        └── Workflow Agent 额外显示：从已创建工作流中选择一个绑定
```

**说明**：
- ReAct Agent 和 Workflow Agent 在同一个创建流程中，通过类型选择分支
- Workflow Agent 的 Step 3 变为"绑定工作流"，不显示工具/知识库选择（这些在工作流画布中配置）
- 编辑页复用创建页的表单组件，预填已有配置

---

### 3. 工作流

工作流的管理和编辑页面。

```
/workflows
├── /workflows                  # 工作流列表页
│   ├── 搜索框
│   ├── 新建工作流 按钮
│   └── 工作流卡片列表（名称、描述、触发方式标签、最近运行时间）
│
└── /workflows/[id]             # 工作流编辑页（全屏画布）
    ├── 顶部工具栏
    │   ├── 工作流名称（可编辑）
    │   ├── 运行按钮（手动触发一次测试）
    │   └── 保存按钮
    │
    ├── 左侧：节点面板
    │   └── 可拖拽的节点列表（Start、End、LLM、If-Else、HTTP Request、
    │       Knowledge Retrieval、Code、Tool、Answer）
    │
    ├── 中间：画布区
    │   ├── 节点和连线
    │   ├── 缩放和平移
    │   └── 右键菜单（删除节点、删除连线）
    │
    ├── 右侧：节点配置面板（点击节点后展开）
    │   ├── 各节点专属配置项
    │   └── 变量引用提示
    │
    └── 底部：运行日志面板（可折叠）
        ├── 运行状态（成功/失败/进行中）
        ├── 各节点的输入输出
        └── 耗时统计
```

**说明**：
- 工作流编辑页是全屏画布，左侧导航可收起
- 触发器配置在工作流编辑页内完成（作为工作流属性配置，不是画布节点）
- 运行日志面板同时用于调试和日常查看

---

### 4. 知识库

知识库的管理页面。

```
/knowledge
├── /knowledge                  # 知识库列表页
│   ├── 搜索框
│   ├── 新建知识库 按钮
│   └── 知识库卡片列表（名称、文档数、chunk 数、最近更新时间）
│
└── /knowledge/[id]             # 知识库详情页
    ├── 顶部：知识库名称 + 描述 + 设置按钮
    │
    └── Tab 切换
        ├── 文档管理 Tab
        │   ├── 上传文档按钮（支持拖拽上传）
        │   ├── 文档列表（文件名、状态：解析中/已完成/失败、chunk 数、操作）
        │   └── 文档详情（点击展开 → chunk 列表，支持编辑 chunk 内容）
        │
        └── 命中测试 Tab
            ├── 查询输入框
            ├── Top-K 数量设置
            ├── 搜索按钮
            └── 结果列表（chunk 内容 + 相似度分数 + 来源文档名）
```

**说明**：
- 知识库创建用弹窗而非独立页面（只需填写名称和描述）
- 命中测试内嵌在知识库详情页中，不独立成页
- 文档解析进度实时更新（解析中 → 已完成 / 失败）

### 5. 工具

工具的管理页面。

```
/tools
├── /tools                      # 工具列表页
│   ├── 搜索框
│   ├── 新建工具 按钮（下拉选择类型）
│   │   ├── 新建 HTTP 工具
│   │   ├── 连接 MCP Server
│   │   └── （Workflow-as-Tool 在工作流发布时自动注册，不在此手动创建）
│   │
│   └── 工具卡片列表
│       ├── 按类型分组：MCP 工具 / HTTP 工具 / 工作流工具
│       └── 每个工具卡片：名称、描述、类型标签、启用/禁用开关
│
├── /tools/create?type=http     # 创建/编辑 HTTP 工具（表单页）
│   ├── 工具名称、描述
│   ├── OpenAPI Schema 上传（可选）
│   └── 手动配置：URL、Method、Header、Body、参数定义
│
└── /tools/create?type=mcp      # 连接 MCP Server（表单页）
    ├── Server 名称
    ├── 连接地址（SSE / Streamable HTTP）
    ├── 认证配置（API Key 等）
    ├── 连接测试按钮
    └── 发现的工具列表（连接成功后自动列出，可逐个启用/禁用）
```

**说明**：
- MCP 工具的粒度是"连接一个 MCP Server"，连接后自动发现其提供的所有工具
- HTTP 工具支持两种创建方式：上传 OpenAPI Schema 自动解析，或手动填写参数
- Workflow-as-Tool 不在此创建——当工作流保存并发布时，自动注册到工具列表中

---

### 6. 模型管理

管理 LLM 模型供应商的连接配置。

```
/models
└── /models                     # 模型管理页
    ├── Provider 列表（已配置的供应商卡片）
    │   ├── 每个卡片：供应商名称、状态（已连接/未连接）、可用模型数
    │   └── 操作：编辑、测试连接、删除
    │
    ├── 添加 Provider 按钮
    │   └── 表单：供应商类型（OpenAI 兼容 / Anthropic）、
    │       API Key、Base URL、模型名称列表
    │
    └── 模型列表（所有已配置模型的总览）
        ├── 模型名称、供应商、类型（LLM / Embedding）
        └── 快速测试按钮
```

**说明**：
- 一期模型管理直接作为一级导航页，不需要二级导航
- 后续如果需要加系统级配置（认证、全局设置等），再考虑增加"设置"导航项

## 用户核心路径

```
首次使用（管理员）：
  模型管理 → 添加模型 Provider
  → 工具 → 连接 MCP Server / 创建 HTTP 工具
  → 知识库 → 创建知识库 → 上传文档
  → 工作流 → 创建工作流（可选）
  → Agents → 创建 Agent（绑定模型/工具/知识库/工作流）

日常使用（所有用户）：
  对话页 → 选择 Agent → 开始对话
  或 → 点击历史会话 → 继续对话
```
